# OpenUI Lang — component reference

Verified signatures from `@openuidev/react-ui@0.11.6`. **Arguments are positional.** `?` marks optional arguments. Optional trailing args may be omitted; intermediate ones must be passed as `null`.

The fully canonical spec is in [spec.txt](spec.txt) (generated by `@openuidev/react-lang`'s `generatePrompt()`). This file is the human-edited companion with verified usage and pitfalls.

## Operators

Any expression on the right side of `name = …` can use these:

| Category | Operators | Example |
|---|---|---|
| Arithmetic | `+`, `-`, `*`, `/`, `%` | `"" + $days + " days"` (string concat with `+`) |
| Comparison | `==`, `!=`, `>`, `<`, `>=`, `<=` | `@Count(data.rows) > 0` |
| Logical | `&&`, `\|\|` | `$loaded && $hasData` |
| Unary | `!`, `-` (negation) | `!$showModal` |
| Ternary | `cond ? a : b` | `$showEdit ? editForm : null` |
| Member | `a.b.c` (dot access; on arrays = column pluck) | `data.rows.title` |

Ternary is the canonical way to conditionally render — wrap a component in `$flag ? Component(...) : null`.

## Layout

| Component | Signature | Notes |
|---|---|---|
| `Stack` | `Stack(children[], direction?, gap?, align?, justify?, wrap?)` | `direction`: `"row"\|"column"` (default column). `gap`: `"none"\|"xs"\|"s"\|"m"\|"l"\|"xl"\|"2xl"`. `align`: `"start"\|"center"\|"end"\|"stretch"\|"baseline"`. `justify`: `"start"\|"center"\|"end"\|"between"\|"around"\|"evenly"`. `wrap`: boolean. |
| `Tabs` | `Tabs(items[])` | `items` is an array of `TabItem`. **No second arg** for controlled state — tab switching is internal. |
| `TabItem` | `TabItem(value, trigger, content[])` | `value` is a unique id string. `trigger` is the visible tab label. `content` is the array of components shown when active. |
| `Accordion` | `Accordion(items[])` | items are `AccordionItem`. |
| `AccordionItem` | `AccordionItem(value, trigger, content)` | `value` unique id. `trigger` is the section title. `content` is a string OR a component. |
| `Steps` | `Steps(items[])` | items are `StepsItem` (note the extra `s`). |
| `StepsItem` | `StepsItem(title, details)` | **Component name is `StepsItem`, not `StepItem`.** Both args required. `details` is descriptive text, NOT a status enum. |
| `Carousel` | `Carousel(items[])` | Horizontal scrollable items; each item is a Card or other content. |
| `Separator` | `Separator()` | Visual divider, no args. |
| `Modal` | `Modal(title, $open, children[])` | `$open` must be a reactive `$boolean`. X / Escape / backdrop click auto-closes via `@Set($open, false)`. Put `Form` with buttons inside. |
| `Card` | `Card(children[], variant?, ...stackArgs)` | `variant`: `"card"` (default elevated) \| `"sunk"` \| `"clear"`. Accepts all `Stack` flex params after variant. Always full-width by default. |

## Content

| Component | Signature | Notes |
|---|---|---|
| `CardHeader` | `CardHeader(title, subtitle?)` | |
| `TextContent` | `TextContent(text, size?)` | `size`: `"small"\|"default"\|"large"\|"small-heavy"\|"large-heavy"`. Supports markdown in text. |
| `MarkDownRenderer` | `MarkDownRenderer(textMarkdown, variant?)` | `variant`: `"clear"\|"card"\|"sunk"`. |

| `TextCallout` | `TextCallout(variant, title, description)` | Lightweight text-only variant of Callout. |
| `Image` | `Image(alt, src?)` | **alt FIRST (required), src optional.** Backwards from typical HTML. |
| `ImageBlock` | `ImageBlock(src, alt?)` | src required, alt optional. Different from `Image`. |
| `ImageGallery` | `ImageGallery(images[])` | items: `{ src, alt?, details? }` object literals — object syntax IS valid inside the array because items are data, not component calls. |
| `CodeBlock` | `CodeBlock(language, code)` | `language`: any syntax-highlighting language id (`"openui"`, `"ts"`, `"js"`, `"py"`, etc.). |

## Forms

Each `FormControl` should be its own top-level reference (not inlined into the Form's array). This enables progressive field-by-field streaming.

| Component | Signature | Notes |
|---|---|---|
| `Form` | `Form(name, buttons, fields[])` | **`buttons` is the 2nd arg** — a single `Button` or a `Buttons(...)` group. `fields` is 3rd. NEVER nest Forms. |
| `FormControl` | `FormControl(label, input, hint?)` | Wraps any input element. |
| `Label` | `Label(text)` | Standalone label. |
| `Input` | `Input(name, placeholder, type?, rules?, value?)` | `type`: `"text"\|"email"\|"password"\|"number"\|"tel"\|"url"`. `rules`: `{required, email, min, max, minLength, maxLength, pattern, url, numeric}` — **rules IS object-literal form**, data not a component call. `value` is the 5th positional — pass a `$variable` here for two-way binding. Use `null` for `rules` if skipping it. |
| `TextArea` | `TextArea(name, placeholder, rows?, rules?)` | |
| `Select` | `Select(name, items[], placeholder?, hint?, value?)` | items are `SelectItem`. `value` is the 5th positional — pass a `$variable` here for two-way binding. Use `null` for skipped middle args. |
| `SelectItem` | `SelectItem(value, label)` | |
| `Slider` | `Slider(name, value, min?, max?, step?)` | `value` typically a reactive `$variable`. |
| `DatePicker` | `DatePicker(name, value?)` | ISO date string. |
| `Calendar` | `Calendar(name, value?)` | Inline calendar; broader than DatePicker. |
| `RadioGroup` | `RadioGroup(name, items[], defaultValue?)` | |
| `RadioItem` | `RadioItem(label, description, value)` | **Three required positional strings.** Schema field order is label → description → value. |
| `CheckBoxGroup` | `CheckBoxGroup(name, items[])` | |
| `CheckBoxItem` | `CheckBoxItem(label, description, name, defaultChecked?)` | `name` is the 3rd positional — NOT the 1st. No `rules` field; validation is on the parent form. |
| `SwitchGroup` | `SwitchGroup(name, items[])` | |
| `SwitchItem` | `SwitchItem(label?, description?, name, defaultChecked?)` | label and description are optional but **`name` is the 3rd positional** — pass `null` for label/description if omitting. `defaultChecked` may be a reactive `$bool`. |
| `ToggleGroup` | `ToggleGroup(name, items[], defaultValue?)` | |
| `ToggleItem` | `ToggleItem(value, label)` | |

## Reactive state

Declare with `$name = defaultValue` at the top of the file:

```openui
$showModal = false
$query = ""
$days = "7"
```

Reference in component args to bind: `Modal("Edit", $showModal, [...])`, `Input("q", "Search...", "text")` then later `result = @Filter(data.rows, "title", "contains", $query)`.

Update via `Action([@Set($showModal, true)])` in button handlers.

Conditional rendering: `$showModal ? Modal(...) : null`.

## Action helpers (the `@` ones)

| Helper | Purpose |
|---|---|
**Action steps** (use inside `Action([...])` on a Button/IconButton):

| Helper | Purpose |
|---|---|
| `@Set($var, value)` | Set a reactive variable. |
| `@Reset($var1, $var2, ...)` | Restore one or more `$variables` to declared defaults. |
| `@Run(ref)` | Execute a Mutation or re-fetch a Query. |
| `@ToAssistant("message")` | Send a message to the host LLM/agent. |
| `@OpenUrl("https://...")` | Open the URL in a new tab. |

**Data builtins** (use anywhere in expressions):

| Helper | Purpose |
|---|---|
| `@Count(array)` | Count elements. Often chained with `@Filter`. |
| `@Filter(array, field, op, value)` | Filter an array. `op`: `"=="`, `"!="`, `">"`, `"<"`, `">="`, `"<="`, `"contains"`. |
| `@Sort(array, field, direction?)` | Sort an array. `direction`: `"asc"` (default) or `"desc"`. |
| `@Each(array, "varname", template)` | Map over rows; loop variable is only inline. |
| `@Sum(numbers[])`, `@Avg(numbers[])`, `@Min(numbers[])`, `@Max(numbers[])` | Standard aggregates. |
| `@First(array)`, `@Last(array)` | First / last element. |
| `@Round(n, decimals?)`, `@Abs(n)`, `@Floor(n)`, `@Ceil(n)` | Number formatters. |

## Pitfall catalog (extended)

Beyond the top-5 in [SKILL.md](SKILL.md):

- **Component-arg objects are silently broken.** `Callout({ variant: "info" })` parses to no error but renders nothing. Always positional.
- **Empty-string `null` confusion.** Pass `null` not `""` when omitting an optional arg in the middle.
- **`Buttons` vs `Button` typo eats actions.** `Buttons` (plural) is the wrapper; `Button` is the leaf.
- **`Tag("Production", "success")` puts "success" in the icon slot.** Use `Tag("Production", null, "sm", "success")`.
- **`Image("https://...", "alt text")` puts the URL in the required `alt` slot and the alt text in optional `src`** — broken icon shows. Use `Image("alt text", "https://...")`.
- **Charts need numeric arrays, not objects.** Use array-pluck: `LineChart(data.rows.day, [Series("Views", data.rows.views)])`.
- **`@Query` and `@Mutation` require a `toolProvider`** to be wired by the host. The Genui extension does NOT provide one in v1 — any spec using `Query(...)` or `Mutation(...)` will render with placeholder/empty data. Stick to static data for prototyping.
- **No nested `Form`s.** Compose with conditional fields instead.

## Source

The canonical spec is regenerated whenever `@openuidev/react-ui` is bumped:

```bash
node --input-type=module -e "
  import('@openuidev/react-ui').then(async m => {
    const lang = await import('@openuidev/react-lang')
    process.stdout.write(lang.generatePrompt(m.openuiLibrary))
  })
" > spec.txt
```

Update this REFERENCE.md anytime new components or signature changes show up in the regenerated spec.
